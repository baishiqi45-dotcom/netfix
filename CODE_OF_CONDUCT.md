# Code of Conduct

This project follows a practical rule: keep collaboration respectful, specific,
and safe for users who may be handling private network credentials.

## Expected Behavior

- Report bugs with concrete reproduction steps.
- Use sanitized proxy and API examples.
- Respect maintainers' requests to remove sensitive logs or screenshots.
- Focus criticism on behavior, code, design, and product impact.

## Not Acceptable

- Posting live proxy passwords, API keys, cookies, bearer tokens, or QR codes.
- Asking contributors to bypass abuse, anti-fraud, account, or geo controls.
- Harassment, threats, or personal attacks.

Maintainers may edit, hide, or remove public content that exposes credentials or
puts users at risk.
