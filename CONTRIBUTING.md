# Contributing to Netfix

Netfix is a local-first macOS network diagnostic and proxy configuration helper.
It may touch proxy credentials, API keys, local reports, and macOS network
settings, so contributions must be careful about privacy and rollback behavior.

## Before Opening A Pull Request

Run the local checks:

```bash
python3 -m pytest -q
python3 scripts/source_export.py --zip --json
python3 scripts/release_audit.py --scope workspace --root open-source-export/Netfix-0.2.0-source
python3 scripts/release_audit.py --scope workspace --root .
cd gui/macos
swift build
```

Before publishing a release package, also run:

```bash
python3 scripts/release_preflight.py --with-dmg-smoke
./scripts/release_gate.sh --strict-workspace
```

The GitHub CI workspace audit is intentionally strict. If old proxy packages,
live credential artifacts, or tracked `Netfix-*.dmg` / `Netfix-*.zip` files are
present, CI should fail instead of silently publishing unsafe source inputs.

Do not attach real proxy URLs, API keys, cookies, bearer tokens, QR codes, or
raw diagnostic reports to issues or pull requests. Use sanitized examples:

```text
proxy.example.com:8000:user:<password>
socks5h://user:<password>@proxy.example.com:1080
sk-...redacted...
```

## Product Rules

- The macOS app is the primary user entry. CLI and HTTP endpoints are for
  advanced users and agents.
- The app must not show internal codes such as `verify_failed` when a plain
  user needs a next step.
- Tier 2 system changes must require explicit user confirmation.
- Proxy credentials must stay in Keychain or in memory, not in logs, shell
  history, screenshots, exports, or release packages.
- Netfix does not sell proxies or promise provider quality.

## Areas To Extend

- Add symptom rules in `rules/symptoms.json`.
- Add service definitions in `rules/services.json`.
- Add proxy-core adapters under `netfix/cores/`.
- Add sanitized real-world cases under `cases/`.

## Release Hygiene

Release artifacts should be generated by the release scripts, not committed to
git. If `git ls-files 'Netfix-*.dmg' 'Netfix-*.zip'` returns anything, remove
those tracked binaries before publishing:

```bash
git rm --cached Netfix-0.2.0.dmg Netfix-0.2.0-macos.zip
python3 scripts/release_audit.py --scope workspace --root .
```

This keeps the local files on disk but removes them from the source release
index.
