# netfix 产品战略与升级路线图

> 版本：v0.1 战略草案  
> 目标：在现有 v0.2 技术原型基础上，明确市场缺口、目标用户、变现路径和下一步落地计划。

---

## 1. 核心判断：我们到底在解决什么问题

netfix 不是又一个代理客户端。代理客户端的市场（Clash Verge Rev、Surge、v2rayN、Shadowrocket、Stash）已经极度拥挤，而且核心能力是**路由流量**。

netfix 的机会是：**当流量已经连不上时，帮用户快速定位并修复问题**。

用一句话说：

> 别人负责「把流量送出去」，netfix 负责「送不出去的时候救回来」。

这个定位有天然优势：
- 不与 Clash/Surge 正面竞争协议和节点。
- 解决的是一个高频、高情绪价值的痛点（"又连不上了！"）。
- 可以做成所有代理客户端的「急救包」，而非替代品。

---

## 2. 市场缺口：用户真正在抱怨什么

### 2.1 典型场景

| 用户原话 | 本质问题 |
|---|---|
| "TG 能连，但 AI 连不上" | 部分流量绕过代理（DNS / QUIC / 规则缺失） |
| "换节点就好了" | 节点 IP 被风控或 GeoIP 判定异常 |
| "网页版能开，App 不行" | HTTP/3 QUIC 或 WebSocket 被拦截 |
| "睡一觉醒来就没网了" | TUN/系统扩展休眠后失效 |
| "开全局也没用" | IPv6 泄漏 / 企业 VPN 冲突 / DNS 污染 |
| "Codex/Claude Code 报错" | CLI 工具没继承系统代理 |

### 2.2 现有工具为什么没解决好

- **Surge**：功能最强，但 $49.99 + 订阅，学习曲线陡峭，适合极客。
- **Clash Verge Rev**：免费开源，但配置复杂，用户常在 GitHub 提 issue "AI 连不上"。
- **Shadowrocket / Stash / Quantumult X**：iOS 起家，Mac 体验是 iPad 移植，不适合生产力场景。
- **v2rayN**：Windows 基因，Mac 体验边缘化。
- **macOS 自带工具**：Wireless Diagnostics、networkQuality 等功能强大但隐藏深，小白找不到。

### 2.3 缺口总结

缺少一个 **macOS 原生、轻量、小白友好、只聚焦「诊断+修复」** 的工具：

1. 自动判断是网络层、DNS、代理还是目标服务本身的问题。
2. 用中文大白话告诉用户原因。
3. 一键执行安全修复（清 DNS、换节点、关 QUIC、禁用 IPv6 等）。
4. 能跟 Kimi / Codex 等 Agent 协作，让 AI 一句话修网。

## 3. 目标用户画像

### 3.1 核心用户：「海外 AI 依赖型 Mac 用户」

- 职业：程序员、设计师、产品经理、自媒体、留学生、自由职业者。
- 日常：每天使用 ChatGPT / Claude / Cursor / GitHub Copilot / YouTube / Twitter。
- 痛点：突然连不上时浪费 10-30 分钟瞎折腾，严重影响工作流。
- 付费意愿：已经为机场/节点付费，愿意为「节省时间」再付一杯咖啡钱。
- 地理：中国大陆、海外华人、跨国远程工作者。

### 3.2 次级用户

- **小白用户**：不懂代理原理，只想"点一下就好"。
- **企业小团队**：团队成员网络问题频发，希望有统一诊断工具。
- **AI Agent 用户**：希望 Kimi/Codex 能自动调用本地工具修网。

## 4. 竞品对标与差异化

| 产品 | 定位 | 优点 | 缺点 | netfix 差异化 |
|---|---|---|---|---|
| Surge Mac | 专业网络工具箱 | 功能极强、协议全 | 贵、复杂、订阅制 | 更便宜、更简单、聚焦修复 |
| Clash Verge Rev | 免费开源代理客户端 | 免费、协议新 | 配置难、 issue 多 | 不替代它，而是它的急救包 |
| Shadowrocket | iOS 代理工具 | 便宜、易用 | Mac 体验差 | 原生 Mac 体验 |
| iStat Menus / Little Snitch | 系统监控/防火墙 | 专业监控 | 不解决代理问题 | 专攻网络连通性 |
| stormzhang/ipcheck | 开源检测脚本 | 检测项全 | 命令行、无修复 | GUI + 自动修复 + Agent |

**核心差异化：**
-  diagnose → reason → fix 的闭环，不是只检测。
- 原生 macOS SwiftUI App，轻量启动快。
- MCP / HTTP API，让 AI Agent 能调用。
- 中文-first 文案，海外华人用户无门槛。

## 5. AI Agent 集成机会

### 5.1 为什么现在做这件事

- MCP（Model Context Protocol）已成为 AI 工具集成的「USB-C 标准」。
- Kimi Code CLI、Codex CLI、Cursor、Claude 都支持 MCP。
- 市场对「本地系统工具被 AI 调用」的需求在增长，但现有 MCP 网络工具多为通用 ping/dns，缺乏 macOS 代理场景的专业修复能力。

### 5.2 netfix 的 Agent 价值

用户可以直接说：

> "我 ChatGPT 打不开了"  
> "帮我看看为什么 Codex 连不上"  
> "网络好像有点慢，诊断一下"

Agent 调用 netfix 后，返回：
- 问题根因（如："ab.chatgpt.com 的 UDP 流量走了直连，规则漏配"）。
- 修复建议或执行修复。
- 最终验证结果。

### 5.3 落地方向

- 继续完善 MCP server，确保 Kimi/Codex 能稳定调用。
- 提供一键安装配置：`kimi mcp add netfix`、`codex mcp add netfix`。
- 在 netfix App 内增加「Ask AI」入口，让不会用 Agent 的用户也能获得解释。

## 6. 商业模式与变现路径

### 6.1 定价锚点

| 竞品 | 定价 |
|---|---|
| iStat Menus 7 | $11.99 买断 |
| Bartender | $15 买断 |
| Little Snitch 6 | $59 买断 |
| Proxyman 个人版 | $89 买断 |
| Surge Mac | ~$49.99 买断 + 升级订阅 |
| Shadowrocket | $2.99 买断 |

### 6.2 推荐模式：买断 + 增值服务

**Personal 版：$19.99 一次性买断**
- 完整诊断 + 修复功能
- 基础服务列表
- MCP / API 支持
- 7-14 天免费试用

**Pro 版：$49/年 或 $6.99/月（订阅）**
- 高级诊断（历史趋势、自动化监控）
- 团队/多设备授权
- 企业 VPN 冲突检测
- 优先规则包更新

**Enterprise 版：按席位报价**
- 集中式审计日志
- MDM 部署
- SSO
- 专属支持

### 6.3 分发渠道

1. **官网直接下载（首选）**：灵活、利润高、可接支付宝/微信/Stripe。
2. **Setapp**：获取全球 Mac 用户曝光，但收入按使用分成。
3. **Mac App Store（谨慎）**：如果产品保持纯诊断、无隧道功能，可以尝试；否则容易被 5.4 条款拒绝或被中国区下架。
4. **GitHub 开源核心**：建立信任、获取早期用户、社区传播。

## 7. 法律与合规边界

### 7.1 中国大陆

- 不可销售或分发 VPN/翻墙服务。
- 不可内置代理服务器、订阅或「智能路由」功能。
- 应避免营销语言如「科学上网」「翻墙」「解锁」。
- 建议：中国大陆用户不主动推广；官网可 Geo-block 大陆 IP。

### 7.2 Apple 平台

- App Store Review Guideline 5.4 对 VPN 类产品审核严格。
- 如果包含 TUN/Packet Tunnel，需要组织开发者账号、正确 NetworkExtension、明确隐私政策。
- 更安全的路线：走 Developer ID 签名 + Notarization 的官网下载。

### 7.3 安全定位

netfix 的法律安全边界：
- **只做本地诊断和修复**。
- **不托管、不销售、不分发代理服务**。
- 用户可以导入自己的代理配置（SOCKS/HTTP），netfix 只帮助检测和切换本地已配置的节点。

## 8. 产品路线图

### Phase A：产品化打磨（1-2 个月）—— 让小白真的能用

目标：从「能跑的技术原型」到「可卖给普通人的产品」。

- [ ] 修复 SwiftUI App 启动和窗口体验，确保双击即用。
- [ ] 设计真正的 onboarding：首次打开自动识别代理客户端、引导用户完成基线检测。
- [ ] 优化中文报告：更少技术术语，更多「人话结论」和「下一步按钮」。
- [ ] 增加「AI 服务急救包」预设：ChatGPT / Claude / Gemini / Copilot / Cursor 域名规则、HTTP/3 检测、IPv6 泄漏检测。
- [ ] 增加 CLI/App 的「一键修复」成功率：Flush DNS、Renew DHCP、Toggle System Proxy、Disable IPv6。
- [ ] 完善 MCP server，通过 Kimi / Codex 实测稳定性。

### Phase B：信任与分发（1-2 个月）—— 让人愿意买

- [ ] 官网落地页 + 中英文双语。
- [ ] 7 天免费试用机制。
- [ ] 支付宝 / 微信 / Stripe 支付接入。
- [ ] GitHub 开源 Lite 核心，保留 Pro 功能闭源。
- [ ] 早期用户案例 / V2EX / 少数派 / Twitter 冷启动。
- [ ] 收集真实 case，持续优化规则库。

### Phase C：商业化升级（2-3 个月）—— 规模化变现

- [ ] 推出 Pro 订阅：历史监控、自动体检、多设备、团队版。
- [ ] Setapp 上架申请。
- [ ] 企业版：MDM、审计日志、SSO。
- [ ] 签名/公证的 `.dmg` 安装包。
- [ ] 自动化更新（Sparkle 或自建）。

### Phase D：生态扩展（长期）

- [ ] Windows 版（基于现有 Python 引擎 + Tauri GUI）。
- [ ] iOS 版（受限更多，可作为 companion app）。
- [ ] 与机场/节点服务商合作分成？（风险高，需谨慎）
- [ ] AI 解释层：集成 LLM 让用户在 App 内直接对话诊断。

## 9. 下一步建议：先落地 Phase A

在 Phase A 中，优先级最高的三件事：

1. **把 App 的启动和首次体验做到傻瓜级**（这是当前最大缺口，用户已经反馈"点击没反应"）。
2. **增加「AI 服务急救包」预设**，让诊断结果对 ChatGPT/Codex 用户有直接价值。
3. **跑一轮真实用户测试**：找 3-5 个非技术朋友，观察他们第一次打开 netfix 的行为，记录卡点。

只有 Phase A 被验证「小白真的能独立使用」，才值得进入 Phase B 的商业化。

## 10. 关键指标（OKR）

**短期（3 个月）：**
- 10 个真实用户完成首次诊断并给出反馈。
- App 从双击到可用 < 5 秒。
- MCP server 在 Kimi 和 Codex 中稳定调用成功率 > 90%。

**中期（6 个月）：**
- 100 个付费用户或 1000 个 GitHub stars。
- 付费转化率 > 5%。
- 收集 50+ 真实 case。

**长期（12 个月）：**
- 月收入覆盖持续开发成本。
- 成为海外 AI 用户群体中知名的「Mac 网络自救工具」。

## 11. 风险清单

| 风险 | 等级 | 应对 |
|---|---|---|
| App Store / 合规风险 | 高 | 不内置代理服务，走官网分发，避免中国区推广 |
| 用户觉得不值 $19.99 | 中 | 免费试用 + 开源 Lite + 持续证明省时间价值 |
| 竞品快速复制 | 中 | 靠社区 case 和 macOS 原生体验建立壁垒 |
| 代理客户端自己加诊断功能 | 中 | 保持中立、跨客户端、Agent 集成差异化 |
| 技术维护成本 | 低 | 保持标准库优先，控制复杂度 |

---

## 12. 结论

netfix 有机会成为 macOS 上第一个真正面向「海外 AI 用户」的**网络自救产品**。

当前技术底座已经具备，但还需要一轮**产品化打磨**才能推向市场。建议先聚焦 Phase A：把小白体验做到位，再逐步引入付费、团队和 Agent 生态。
