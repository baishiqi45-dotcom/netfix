# 2026-06-29 - 普通用户无法独立完成代理部署

- 触发场景：首次打开 Netfix macOS App，目标是把购买的住宅/HTTP/SOCKS 代理应用到这台 Mac 上网。用户不懂命令行、代理协议、Keychain、本地桥接、127.0.0.1。
- netfix 关键输出：首页出现「本地网络 / DNS 层 / 代理层 / 出口身份 / 路径/质量 / 目标服务」等状态卡片，以及「AI 服务急救包」；代理入口文案使用「部署代理」「预检」「保存并检测」「接管系统流量」「部署到这台 Mac」等技术术语；ProxySetupView 提示「不是只复制出口 IP」，但未在输入框实时校验或给出足够示例；保存后提示「还没接管系统流量」，需再到已保存代理列表点「部署到这台 Mac」才真正生效。
- 根因：产品文案和流程按工程师视角设计，术语密集、两步操作（保存 vs 部署）语义断层明显、缺少面向普通用户的示例与实时校验，导致用户容易在「复制什么」「保存后为什么还不能上网」「要不要点部署」这几个环节放弃。
- 实际修复（建议）：
  1. 首页弱化专业术语，把「部署代理」改成「让这台 Mac 用代理上网」。
  2. 输入框给出明确 placeholder 和示例按钮，粘贴不完整时提示具体缺了什么（用户名/密码/地址/端口）。
  3. 把「保存并检测」改为「保存并测试（暂不影响系统）」，把「部署到这台 Mac」改为「开始用这台 Mac 上网」，并用更醒目的流程区分。
  4. 菜单栏与退出弹窗去掉「桥接」「转发」「系统流量」等词，统一用「代理：开/关/异常」。
  5. 首页减少 AI 功能视觉权重，避免普通用户误以为必须先配 AI。
- 验证：以普通用户视角重走一遍首次启动 → 粘贴代理信息 → 测试 → 应用到本机 → 浏览器可上网，全程无需理解 SOCKS/HTTP、Keychain、桥接等概念。
