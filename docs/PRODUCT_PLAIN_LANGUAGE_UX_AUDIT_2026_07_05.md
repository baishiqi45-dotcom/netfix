## 一、人话判断：现在 Netfix 像不像普通人能用的产品？

**结论：比原来像多了，但还没完全像。**

原来最大的问题不是功能，而是**工程师视角太浓**：首页同时摊出四张状态卡片、四个 AI 快捷按钮、一个「基线检测」大按钮、一个「自动处理低风险问题」开关，普通用户第一眼不知道先点哪个。README 首屏虽然口号不错，但下面紧跟着 MCP、CLI、API、AI 配置，把普通用户吓退。

改完之后：
- 首页层级变成「状态条 → 粘贴代理 → 一键诊断 → 常用网站 → 日志/AI」，主入口清楚。
- 设置页从 6 个 Tab 压缩成「普通设置 / 诊断与日志 / 高级与开发者」，AI 和 MCP 不再跟普通设置平级。
- 错误文案基本去掉「后端」「代理核心」「基线」等词，改成「本地服务」「代理软件」「快照」。
- README 首屏先说「粘贴参数 → 开始使用 → 恢复」，再说开发者入口。

但还有几个硬伤：**App 仍是 QA 未签名版**（这是客观限制，必须继续强调）、**没有真实截图/GIF**、**代理部署成功后对用户来说「Netfix 必须保持打开」这件事说得还不够直白**。

如果 1-10 分打分：
- 改之前普通人可用度：4/10
- 改之后普通人可用度：7/10
- 想上 Product Hunt / 少数派 / 小红书：还需要签名版 + 6 张以上真实截图，才能到 8/10

## 二、用普通用户语言重写 Netfix 的一句话定位

**新版一句话定位：**

> 买了代理不会配 Mac？把服务商后台那串参数粘贴进 Netfix，先检查能不能用，再一键开始使用；不用时也能恢复原来的网络设置。

这句话同时回答了：
1. 我是谁（Mac 用户）
2. 我有什么（服务商给的参数）
3. 我要做什么（粘贴）
4. 我会得到什么（能用 / 能恢复）
5. 风险可控（不用时能恢复）

## 三、20 个「不说人话 / 不干人事 / 用户会懵」的点

以下问题基于本次审核前的版本，**已直接修改大部分**。

### 首页 / Dashboard

1. **首页同时出现「一键诊断」「处理建议」「检查常用网站」「AI 快捷按钮」「自动处理低风险问题」五个操作区**，没有主次，用户不知道第一步该点哪个。
2. **「处理建议」按钮常驻但常处于禁用状态**，用户会反复尝试点击，然后觉得自己操作错了。
3. **「自动处理低风险问题」开关放在首页底部工具栏**，普通用户连「低风险问题」是什么都不知道。
4. **AI 快捷按钮区四个大按钮平铺**，标题「下一步想干嘛？」太散漫，把 AI 捧得比核心功能还重要。
5. **状态卡片默认展开「查看技术细节」**，普通用户会以为自己必须看懂这些才能继续。
6. **顶部状态条显示「正在准备 Netfix…」「Netfix 异常」等工程状态**，没有翻译成「正在启动…」「服务还没准备好」。
7. **代理部署入口和诊断/AI 入口同级**，导致「粘贴代理」这个核心动作不突出。

### 代理部署 / ProxySetup

8. **首次启动页标题「添加你的代理」下面，先显示「客户端：xxx / 当前节点：xxx / 端口：xxx」**，像系统信息面板，不像引导。
9. **「运行基线检测」按钮又大又显眼**，首次用户会以为必须先跑这个才能继续。
10. **参数类型 Picker 显示「socks5h」**，普通人不知道 h 是什么意思。
11. **输入框 placeholder 堆了三行**：格式示例、协议说明、不支持格式，信息过载。
12. **「只检查，不保存」和「检查并保存到这台 Mac」两个按钮并列**，用户会纠结选哪个。
13. **「开始使用这台 Mac 上网」按钮文案太长**，而且出现在多个位置，口头禅化。
14. **部署确认弹窗文案长但没说清「Netfix 需要保持打开」**，用户可能关 App 后发现网络断了。

### 设置 / Settings

15. **设置页有 6 个 Tab：通用、部署代理、AI 编程助手、AI、权限、关于**，普通用户感觉进了配置中心。
16. **「AI 编程助手」Tab 名字像给程序员用的，跟普通 Mac 用户没关系**，却放在显眼位置。
17. **「服务分组」这种工程概念直接暴露在设置页**，用户不知道怎么用。
18. **代理设置里「高级：服务分组 / 启动项 / 高级代理控制」嵌套 DisclosureGroup**，层级太深。

### 错误文案

19. **错误提示里出现「后端要求登录」「后端没找到这条」「App 与后端版本对不上」**，用户会问「后端是什么」。
20. **恢复路径写「到部署代理 → 更多 → 恢复原来的网络设置」**，但新版 UI 已经没有「部署代理」这个 Tab，路径对不上。

## 四、最伤产品力的 10 个问题

按杀伤力从高到低：

1. **核心入口被淹没**：首页没有让「粘贴代理」成为绝对主角，导致用户第一步就迷路。
2. **设置页像配置中心**：6 个 Tab + AI 编程助手平级，普通用户不敢点。
3. **首次启动强推「基线检测」**：用户还没理解 App 是干嘛的，就被要求跑检测。
4. **错误文案泄露出工程身份**：「后端」「代理核心」「基线」让用户觉得这不是给自己用的。
5. **恢复路径描述与 UI 不符**：用户真需要恢复时找不到按钮。
6. **AI/MCP 抢了普通用户入口**：README 和设置页都把开发者功能放在太前面。
7. **没有真实视觉资产**：README 只有占位图，GitHub 首屏缺少截图/GIF，star 转化率天然低。
8. **部署成功后的约束没说清**：Netfix 本机转发需要保持 App 打开，否则系统代理会指向一个死端口。
9. **「处理建议」按钮常驻禁用**：破坏用户信心，也浪费首屏空间。
10. **socks5h / PAC / WPAD / mixed 端口等术语外露**：每一步都在提醒用户「你不懂」。

## 六、首页 / 代理部署页 / 错误提示 / 设置页优化方案

### 首页优化方案
- **当前状态条永远置顶**，用一句话告诉用户现在该做什么。
- **「粘贴代理参数」作为首页第一动作**，其他全部次要。
- **「一键诊断」只在没有明确问题时作为推荐动作出现**。
- **AI 入口收缩到「看不懂诊断结果？」一个小入口**。
- **底部只保留「诊断」「代理」「日志」「设置」四个入口**。

### 代理部署页优化方案
- **首屏只问一个问题：「你有代理参数吗？」**
- **输入框上方用一张小图/示例说明「去服务商后台复制这一行」**。
- **只保留一个主按钮：「检查并保存」**，「只检查」作为辅助链接。
- **保存成功后直接出现「开始使用代理」按钮**，不要让用户再回设置页找。
- **部署成功后明确提示：「请保持 Netfix 打开；退出前请先恢复网络设置」**。

### 错误提示优化方案
- **每个错误 = 一句话结论 + 一个具体下一步 + 一个「复制这段说明」按钮**。
- **不再默认展示技术详情**，折叠到「查看技术详情」。
- **避免「可能」「建议」「请检查」堆叠**，改成「地址抄错了，重新粘贴」「节点连不上，换一个备用节点」。
- **恢复入口跟错误提示同时出现**，不要让用户自己去找。

### 设置页优化方案
- **只保留三层：普通设置 / 诊断与日志 / 高级与开发者**。
- **普通设置里放：启动、通知、代理粘贴/恢复、权限、关于**。
- **AI/MCP 全部放进「高级与开发者」**。
- **每个开关配一句人话说明**，不要只有标签。

## 七、GitHub README 首屏优化方案（提高 star 转化）

### 首屏 5 秒测试
用户打开仓库 5 秒内必须看到：
1. 这是什么：帮 Mac 用户粘贴代理参数并一键使用的小工具。
2. 能解决什么问题：买了代理不会配 Mac / Codex GitHub 连不上 / 代理开了但不生效。
3. 怎么试用：一行命令安装，dry-run 先看效果。
4. 为什么可信：本地优先、密码进 Keychain、改前备份、随时恢复。

### 首屏结构
1. **Hero 图**：App 主界面截图，状态条显示「代理已保存，点开始使用代理」。
2. **一句话 slogan**（见第二节）。
3. **三句话介绍**：粘贴 → 检查 → 使用 → 恢复。
4. **三步安装/使用**：下载 → 粘贴 → 开始。
5. **一排信任徽章**：MIT、macOS、本地优先。
6. **对比表**：跟 Clash/Surge/Activity Monitor 的区别，一句话说完。
7. **「如果失败怎么恢复」放在首屏下半部分**，降低决策门槛。
8. **开发者入口放在折叠区或第二节**，不要抢首屏。

### star 转化关键
- **必须有真实截图**：没有截图的 GitHub 项目，star 转化率至少掉 50%。
- **案例前置**：把 `cases/2026-06-29-普通用户代理部署体验审查.md` 的痛点摘要放在 README「它解决什么问题」前面。
- **诚实说限制**：未签名 QA 版要说清楚，这比假装正式版更能获得技术用户信任。
- **CTA 具体**：不要写「欢迎 star」，要写「如果它帮你定位过一次 Mac 网络问题，右上角点个 ⭐」。

## 十、下一轮最值得做的 5 个功能/设计改动

1. **首屏「粘贴代理」向导化**
   把 ProxySetup 做成覆盖层/向导，用户打开 App 空状态直接看到：
   - 去服务商后台复制这一行（示例图）
   - 粘贴到这里
   - 点检查
   - 点开始使用
   不要让用户自己发现入口。

2. **部署成功后强制提示「保持 App 打开」**
   当用户使用本机转发模式时，弹出一个不可跳过的提示：
   「 Netfix 现在正在帮你代管代理密码。如果退出 App，网络会断开；请先点『恢复原来的网络设置』再退出。」

3. **错误卡片直接附带恢复按钮**
   当检测到需要恢复网络设置时，错误卡片上直接出现「恢复原来的网络设置」按钮，而不是让用户去设置页找。

4. **为代理服务商做「复制这一行」引导图**
   在粘贴区旁边放一张示意图：
   - 服务商后台常见字段：地址 / 端口 / 用户名 / 密码
   - 不要复制：当前出口 IP、ss:// 链接、Clash 订阅
